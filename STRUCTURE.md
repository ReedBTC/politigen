# GenGov Repository Structure

```

│   │                                #            age, generation, generation_short

│   │                                #   (1901–2025). One row per person per Congress.
│   │                                #   Columns: congress, congress_year, name,
│   │                                #            chamber, state, party, date_of_birth,
│   │                                #            birth_year, age_at_start, generation

│   │                                #   Age bracket distributions by industry sector.
│   │                                #   NOTE: Retail Trade and Transportation rows
│   │                                #   excluded (bad source matches).
│   │                                #   Refresh: python scripts/collect_bls_workforce.py
│   │

│                                     # Used for BLS vs. officials comparison chart.
│                                     # Audit trail for chart_bls_by_gen.html

├── scripts/                         # Data collection and chart generation
│   │
│   ├── collect_officials.py         # Pulls current Congress from unitedstates/

│   │                                #   Outputs: data/officials.csv
│   │
│   ├── collect_historical.py        # Pulls all Congresses 57th (1901)–119th (2025).
│   │                                #   Outputs: data/historical_congress.csv

│   │
│   ├── collect_bls_workforce.py     # Pulls and parses BLS Table 18b Excel file.

│   ├── make_charts.py               # Generates charts 1–5:

│   │
│   ├── chart_bls_comparison.py      # Generates BLS age-bracket vs officials chart
│   │
│   └── chart_bls_by_gen.py          # Generates generation-share BLS vs officials

│
├── charts/                          # All output charts (HTML — open in any browser)
│   │
│   ├── chart_historical_age_trend.html   # Congress mean/median age 1901–2025
│   │                                     # THE core aging visualization
│   │
│   ├── chart_gen_rise_fall.html          # Rise & fall of each generation in Congress
│   │                                     # Full Congress + Senate panel

│   ├── chart_aging_in_place.html         # Snapshot comparison: 1965/1985/2005/2025

│   │
│   ├── chart_donut.png                   # Generational breakdown donut (current)

    ├── index.html                   # Landing page — links to all charts
    └── [charts copied here for      # GitHub Pages serves from /docs
```

## Data Flow

```
GitHub API (unitedstates/congress-legislators)
    └── collect_officials.py ──────► officials.csv
    └── collect_historical.py ─────► historical_congress.csv
                                     congress_snapshots.csv

BLS.gov (CPS Table 18b Excel)
    └── collect_bls_workforce.py ──► bls_workforce_ages.csv

officials.csv + bls_workforce_ages.csv
    └── chart_bls_by_gen.py ───────► chart_bls_generation_data.csv
                                     chart_bls_by_gen.png/html

congress_snapshots.csv
    └── make_charts.py ────────────► chart_*.png
    └── [manual HTML charts] ──────► chart_*.html
```

## Generational Framework

All generation assignments use **Strauss-Howe** cutoffs (not Pew):

| Generation       | Birth Years | Notes                        |
|------------------|-------------|------------------------------|
| Silent           | 1925–1942   |                              |
| Baby Boom        | 1943–1960   |                              |
| Gen X (13ers)    | 1961–1981   |                              |
| Millennial       | 1982–2005   |                              |
| Gen Z            | 2006+       | No current federal officials |

See `methodology.md` for full rationale and edge case decisions.

## Refreshing the Data

```bash
# Set up environment
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Refresh all source data
python scripts/collect_officials.py
python scripts/collect_historical.py
python scripts/collect_bls_workforce.py

# Regenerate charts
python scripts/make_charts.py
python scripts/chart_bls_comparison.py
python scripts/chart_bls_by_gen.py
```

The HTML charts in `charts/` are standalone files — no server needed,
just open in a browser. They embed all data directly in the file.
